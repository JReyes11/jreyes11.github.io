Building Google Chrome extensions

Context:

In my previous role, I worked with a number of collegues outside of engineering who were frequently blocked when needing to create user accounts and perform any and all actions in the lower test environments. This reoccurance inspired me to create a UI that would perform the desired actions and required minimal user input.

Ultimately I decided to build a google chrome extension, ( as opposed to using Express + Node.JS ) allowing for interactions with several dropdowns, radio buttons and checkboxes. From the quality perspective, unblocking my non technical collegues was a return on investment, as it utlimately led to exploratory testing efforts.

In addition:

  • Option to generate random user account or specify desired username.
  • Option to select from a collection of products that are to be registered to the generated user.
  • Once options have been selected, a series of requests were made to AWS Lamdas and other internal backend services.
  • Information tab that explains how the tools works, along with how to perform these steps manually if desired.
  • Information on how to validate the expected datasets in the rails console and other third party applications.

Achievements:
  • Tool was widely used cross functionally by Developers, Product owners and Quality Engineers.
  • Periodically received feature requests which led to in-depth conversations and knowledge sharing.
